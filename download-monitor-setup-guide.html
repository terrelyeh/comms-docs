Download Monitor 設定指南

追蹤 EPC Downloads 頁面的檔案下載次數

建立日期：2026-03-25  |  適用網站：www.engeniustech.com

1

為什麼需要這個外掛？

我們需要知道 Downloads & Guides 頁面上的檔案（例如 EPC_Installer.exe）被下載了幾次。

雖然網站有安裝 GA4（透過 GTM），但 GA4 的 file_download 事件沒有正確收到檔案名稱參數，無法區分是哪個檔案被下載。

Download Monitor 是一個免費的 WordPress 外掛，可以在後台直接看到每個檔案的精確下載次數。

2

運作原理

Download Monitor 的原理很簡單——它在使用者和實際檔案之間加了一層「計數器」。

改之前（無法追蹤）

使用者點「Download」

↓

直接連到 S3 下載檔案

沒有任何紀錄

改之後（有追蹤）

使用者點「Download」

↓

經過 Download Monitor（計數 +1）

↓

自動轉到 S3 下載檔案

每次下載都有紀錄

💡
使用者感受不到差異 下載體驗跟以前完全一樣，只是連結 URL 變了。檔案不需要重新上傳或搬移，仍然放在原本的 S3 上。
3

設定步驟

Step A：在 Download Monitor 建立下載項目

  1. 登入 WordPress 後台 /wp-admin
  2. 左側選單 → Downloads → Add New
  3. 在標題欄填入辨識名稱，例如：EPC Installer v1.8.8 (Windows)
  4. 往下找到 「Downloadable Files/Versions」 區塊
  5. 點 「Add file」
  6. 在 File URL 欄位貼上原本的 S3 下載連結，例如：
    https://engenius-epc.s3.us-west-2.amazonaws.com/dev/1.8.8/epc.sh
  7. 點 Publish（發佈）
  8. 發佈後記下這個項目的 ID（網址列會顯示 post=123）
⚠️
不需要上傳檔案 只要貼上現有的 S3 URL 即可，檔案繼續放在 S3 上，不需要搬到 WordPress。

Step B：取得新的下載連結

每個建好的下載項目，Download Monitor 會自動產生一個追蹤連結：

// 格式如下（123 換成你的 ID）
https://www.engeniustech.com/?download=123

// 如果有設定 permalink，也可能是
https://www.engeniustech.com/download/epc-installer-v1-8-8/

Step C：替換頁面上的連結

  1. 到 Downloads & Guides 頁面的編輯畫面
  2. 找到原本的下載按鈕/連結
  3. 將原本的 S3 直連：
    https://engenius-epc.s3...amazonaws.com/dev/1.8.8/...
    替換成 Download Monitor 的連結：
    https://www.engeniustech.com/?download=123
  4. 儲存頁面
  5. 測試點擊，確認能正常下載檔案

哪些檔案需要設定？

只需要針對想追蹤的檔案操作，不需要把所有下載都搬進來。

檔案 平台 需要追蹤？ 動作
EPC_Installer.exe v1.8.8 Windows 由你決定 加進 Download Monitor + 換連結
EPC_Installer.exe v1.8.7 Windows 由你決定 如不需要可跳過
epc.sh v1.8.8 Linux 由你決定 加進 Download Monitor + 換連結
epc.sh v1.8.7 Linux 由你決定 如不需要可跳過
epc.sh v1.7.2 (beta) Linux 由你決定 如不需要可跳過
✅
建議 先針對最新版本（v1.8.8 的 Windows + Linux）做就好，未來新版本上架時再新增。
5

如何查看下載次數

  1. WordPress 後台 → 左側 Downloads
  2. 在列表中每個檔案旁邊會顯示 Download Count 欄位
  3. 點進單一檔案也可以看到更詳細的下載記錄

如果需要更進階的報表（圖表、匯出），那是 Pro 付費版功能，但目前免費版的計數已經夠用。

6

注意事項

🚨
換連結後一定要測試 替換完畢後，務必用無痕視窗點擊下載按鈕，確認：(1) 檔案能正常下載、(2) 回到 Downloads 頁面看計數有 +1。
⚠️
多語系 / 多區域頁面 網站是全球共用 domain（/tw、/eu 等），如果不同地區頁面都有同一個檔案的下載連結，全部都要換成 Download Monitor 的連結才能完整追蹤。一個 Download Monitor 項目可以在多個頁面使用。
⚠️
未來新版本上架時 每次有新版本的 EPC Installer，除了上傳到 S3 之外，也要在 Download Monitor 新增一筆對應的下載項目，並更新頁面連結。建議加進 SOP。
💡
Linux 的 Deploy Command Linux 區域顯示的是 wget 指令，使用者是複製指令後在終端執行，不是點擊下載。這種情況 Download Monitor 無法追蹤（因為使用者直接連到 S3）。如果要追蹤 Linux 的下載，需要把 wget 的 URL 也換成 Download Monitor 的連結。
7

設定完成 Checklist

  • Download Monitor 外掛已安裝並啟用
  • 已為需要追蹤的檔案建立 Download 項目
  • 每個項目都已貼上正確的 S3 檔案 URL
  • 頁面上的下載連結已替換為 Download Monitor 連結
  • 用無痕視窗測試過，確認可正常下載
  • 確認 Download Count 有正確 +1
  • 如有多語系頁面（/tw、/eu），各頁面的連結都已更新
