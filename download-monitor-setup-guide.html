檔案下載追蹤方案

EnGenius 官網 Downloads & Guides — 如何追蹤檔案下載次數

2026-03-25

Executive Summary

我們需要追蹤官網檔案的下載次數，但目前沒有任何記錄機制。

問題：GA4 有收到下載事件，但因為檔案放在外部 S3，無法辨識是哪個檔案被下載。

挑戰：全球 5 個站是獨立 WordPress 主機，各自埋了不同的 GTM container。

建議：採混合方案——全部 5 個站都安裝 Download Monitor 外掛立即上線，同時盤點各站 GTM/GA4 狀況，後續統一用 GA4 追蹤全球數據並整合報表。

短期方案
5 站皆安裝 Download Monitor（免費）
長期方案
GTM 自訂事件 → GA4 統一報表
立即行動
各站安裝外掛 + 替換下載連結
現況

網站架構與 GTM / GA4 狀態

全球共用 www.engeniustech.com，用資料夾區分區域，但背後是 5 個獨立的 WordPress 主機。各站埋了不同的 GTM container：

www.engeniustech.com ├── / 美國（全球） GTM-WJM42FW GA4: G-3QS0NVS8NH ├── /tw/ 台灣 GTM-WJM42FW GA4: G-3QS0NVS8NH ← 相同 ├── /apac/ APAC GTM-WPWPVMZ GA4: 待確認 ├── /in/ 印度 GTM-5PJ42BPL GA4: 待確認 └── /eu/ 歐洲 GTM-MBP47LB GA4: 待確認（使用變數）
問題

為什麼目前追蹤不到？

GA4 有收到 file_download 事件（三月份約 1.4K 次），但 file_name 參數是空的——知道有人下載，但不知道下載了什麼。原因是下載連結指向外部 S3 網址，GA4 的 Enhanced Measurement 無法正確擷取檔案名稱。

方案比較

三種追蹤方案

方案 A

Download Monitor 外掛

每個站各自安裝 WordPress 外掛

適合：只追蹤 1～2 個站
優點：設定簡單，數據精確
缺點：多站要看多個後台，無法統一報表
費用：免費
方案 B

GA4 統一追蹤

在各站 GTM 加自訂事件，統一送到 GA4

適合：全部 5 站統一追蹤
優點：一個畫面看全球數據，不用改頁面
缺點：需要 4 個 GTM 的權限、設定較複雜
前提：確認各站 GA4 指向同一個 Property
方案 C — 推薦

混合方案

全部 5 站裝外掛快速上線，再推進 GA4 統一追蹤

適合：先追蹤台灣站，未來擴展全球
優點：馬上有數據，不用等全部搞清楚
缺點：短期內台灣站要看 WP 後台
兩邊互不衝突，可並行
建議

推薦方案 C 的執行計畫

Phase 1
本週
全部 5 站安裝 Download Monitor，立即開始收集數據

在所有站（/、/tw/、/apac/、/in/、/eu/）的 WordPress 安裝 Download Monitor，將想追蹤的檔案加入外掛，替換頁面連結。

Phase 2
1～2 週內
盤點其他站的 GTM / GA4 狀況

確認 APAC、印度、歐洲站的 GA4 Measurement ID 是否與美國/台灣站相同（G-3QS0NVS8NH），以及是否有 GTM 管理權限。

Phase 3
盤點後
在 GTM 建立自訂下載事件，GA4 統一查看全球數據

在各 GTM container 加 Click Trigger + GA4 Event Tag，將檔案名稱等資訊送到 GA4，實現一個畫面看全球下載數據。

Phase 1 操作

Download Monitor 設定指南

原理：在使用者與 S3 檔案之間加一層計數器。檔案不用重新上傳，只需要貼 S3 URL，然後把頁面上的連結換成外掛產生的追蹤連結。使用者下載體驗不變。

範圍：全部 5 個站（/、/tw/、/apac/、/in/、/eu/）都要執行以下步驟。每個站是獨立的 WordPress，所以每個站都要各自安裝外掛、建立下載項目、替換連結。

每個站的操作步驟

  1. 登入該站的 WordPress 後台，安裝並啟用 Download Monitor 外掛
  2. 左側選單 Downloads → Add New
  3. 標題填入辨識名稱，例如 EPC Installer v1.8.8 (Windows)
  4. 在 Downloadable Files/Versions 區塊點 Add file，貼上 S3 URL
  5. 點 Publish，記下項目 ID
  6. 回到下載頁面，把原本的 S3 直連替換成：https://www.engeniustech.com/?download={ID}
  7. 用無痕視窗測試，確認能下載且計數有 +1
⚠️
只做想追蹤的檔案就好 不需要把所有下載都搬進來。建議先做最新版（v1.8.8）的 Windows + Linux 即可。
💡
Linux wget 指令的限制 Linux 區域顯示的是 wget 指令，使用者在終端執行不經過瀏覽器。Download Monitor 和 GA4 都無法追蹤這種行為，除非把 wget URL 也換成追蹤連結。
