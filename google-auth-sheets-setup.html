Google 登入 +
Sheets API 設定指南

說明內部工具串接 Google Login（白名單制）與 Google Sheets API 時，需要在 Google Cloud Console 和 Vercel 做哪些設定。

2026-04-05 ・適用於 Next.js + Vercel 部署架構

架構總覽

我們的營收儀表板需要兩種 Google 服務，用了兩套獨立的認證機制：

Google Cloud Console
├── OAuth Client ID（前端登入用）
│   使用者在瀏覽器登入 Google
│   → 取得 ID Token（證明「我是誰」）
│   → 後端驗證 + 白名單比對
│
└── Service Account（後端 API 用）
    伺服器用 private key 取得 access token
    → 直接呼叫 Sheets API 讀寫資料
    → 不需要使用者介入
OAuth Client IDService Account
誰在用使用者（前端瀏覽器）伺服器（後端 API）
做什麼驗證「這個人是誰」讀寫 Google Sheet
是否 Secret❌ 不是（前端公開）✅ 絕對保密
在哪裡設定Credentials → OAuthIAM → Service Accounts
互相依賴嗎不依賴不依賴
兩者搭配的效果：OAuth 確保只有白名單的人能操作，Service Account 確保操作能實際寫進 Sheet。

Google Cloud Console 設定

🔑

OAuth Client ID

讓使用者用 Google 登入

OAuth Client ID 讓你的網站可以顯示「Sign in with Google」按鈕。使用者點擊後，Google 回傳一個 ID Token（JWT 格式），裡面包含 email、名字、頭像。

設定步驟

  1. 進入 Google Cloud Console
  2. 選擇你的專案（或建立新專案）
  3. 前往 APIs & Services → Credentials
  4. 點 Create Credentials → OAuth client ID
  5. Application type 選 Web application
  6. 在 Authorized JavaScript origins 加入你的網域
  7. 建立後取得 Client ID
注意：授權網域

本地開發和正式環境的網域都要加，例如 http://localhost:3001 和 https://your-app.vercel.app。

是否為 Secret❌ 不是。Client ID 會出現在前端程式碼中，本來就是公開的
後端也需要嗎✅ 後端驗證 ID Token 時，需要 Client ID 來確認 token 是發給你的應用
白名單怎麼做白名單不在 Google 設定，是在後端用 ALLOWED_EMAILS 環境變數控制
🤖

Service Account

讓伺服器讀寫 Google Sheet

Service Account 是一個「機器人帳號」，有自己的 email 和 private key。伺服器用它來代表應用程式存取 Google API，完全不需要使用者登入或授權。

設定步驟

  1. 前往 IAM & Admin → Service Accounts
  2. 點 Create Service Account，取名建立
  3. 進入該帳號 → Keys → Add Key → Create new key → JSON
  4. 下載金鑰 JSON 檔案（妥善保管！）

還要做的事

  • 啟用 Google Sheets API：APIs & Services → Library → 搜尋「Google Sheets API」→ Enable
  • 共用 Sheet 給 Service Account：打開 Google Sheet → 共用 → 貼上 Service Account email → 權限選「編輯者」
安全警告

Service Account 的 private key 等同密碼，外洩就等於任何人都能讀寫你的 Sheet。絕對不要 commit 到 Git 或分享給他人。

是否為 Secret✅ 是！private key 絕對保密
跟使用者有關嗎❌ 完全無關。是伺服器對伺服器的認證
需要使用者授權嗎❌ 不需要。Sheet 有共用給 Service Account email 就能存取

Vercel 環境變數設定

在 Vercel Dashboard → 你的專案 → Settings → Environment Variables 設定以下變數：

變數名值說明
GOOGLE_SERVICE_ACCOUNT_KEY 整份 JSON 金鑰內容 下載的 JSON 檔全文貼上
GOOGLE_SHEET_ID Sheet 網址中的 ID /d/ 和 /edit 之間那段字串
GOOGLE_CLIENT_ID OAuth Client ID 格式：xxxxx.apps.googleusercontent.com
ALLOWED_EMAILS email 清單 逗號分隔，例如 alice@gmail.com,bob@gmail.com
本地開發：在專案根目錄建立 .env.local 檔案，貼上同樣的變數。這個檔案已被 .gitignore 排除，不會被 commit。

常見問題

Q：GOOGLE_SERVICE_ACCOUNT_KEY 太長，Vercel 貼得進去嗎？
可以。Vercel 支援多行值，直接把整份 JSON 貼上即可。
Q：private_key 中的 \n 會不會有問題？
有時候會。如果部署後 API 報錯 error:0909006C:PEM routines，通常是 \n 被 escape 成 \\n。確認 Vercel 環境變數中的換行是真正的換行。
Q：白名單要改怎麼辦？
修改 Vercel 上的 ALLOWED_EMAILS 環境變數，然後 Redeploy 即可生效。不需要改程式碼。
Q：OAuth Client ID 和 Service Account 是同一個東西嗎？
不是。它們是完全獨立的兩套認證機制。OAuth Client ID 負責使用者登入，Service Account 負責伺服器讀寫 API。兩者不互相依賴。

設定清單

📋

Google Cloud Console

一次性設定
  • 建立 GCP 專案（或使用既有專案）
  • 啟用 Google Sheets API
  • 建立 OAuth Client ID（Web application），加入授權網域
  • 建立 Service Account，下載 JSON 金鑰
  • 在 Google Sheet 共用設定中，加入 Service Account email 為編輯者
▲

Vercel

環境變數設定
  • 設定 GOOGLE_SERVICE_ACCOUNT_KEY（JSON 全文）
  • 設定 GOOGLE_SHEET_ID（Sheet ID）
  • 設定 GOOGLE_CLIENT_ID（OAuth Client ID）
  • 設定 ALLOWED_EMAILS（白名單 email）
  • 部署並測試登入和資料讀取
