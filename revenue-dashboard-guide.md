# MINGCHU 營收儀表板使用指南

> 建立日期：2026-04-05
> 適用對象：業務、PM、管理者

---

## 一、概述

MINGCHU 營收儀表板將 Google Sheets 的營收追蹤數據轉換為互動式儀表板，讓團隊在週會 / 月會中一眼掌握業績達成率、Pipeline 健康度與各季差距分析。

本指南說明資料填寫規則、各欄位之間的運算邏輯，以及 Best Practice。

---

## 二、Sheet 欄位說明與填寫方式

### 營收專案列表欄位（每一列 = 一筆收款紀錄）

| 欄位 | 說明 | 填寫方式 |
|---|---|---|
| **A. 類型** | 收入的確定性分類 | 從四個選項選一：現金收入 / 預計現金收入 / 預估現金收入 / 資源交換 |
| **B. 月份** | 預計入帳月份 | 填數字 1–12；尚未確定填 0（儀表板顯示「待定」） |
| **C. 名稱** | 專案 / 合作案名稱 | 完整名稱，方便搜尋 |
| **D. 客戶** | 客戶名稱 | 統一用全名（例如「聯合利華」，不要有時寫「UL」） |
| **E. 規模** | 專案金額規模分級 | A（0–10 萬）/ B（11–30 萬）/ C（31–60 萬）/ D（61–100 萬）/ E（100 萬+） |
| **F. 業務** | 負責業務 | 填人名 |
| **G. PM** | 負責 PM | 填人名 |
| **H. 類別** | 專案類別 | 依現有分類填寫 |
| **I. 自辦/外部** | 執行方式 | 自辦 / 外部 |
| **J. 分期** | 付款分期描述 | 全額 / 頭款 / 尾款 / 2-1 / 3-2 等 |
| **K. 未稅金額** | 未含稅金額 | 填數字（元），不加 $ 或逗號 |
| **L. 含稅金額** | 含 5% 營業稅 | = 未稅 × 1.05 |
| **M. 專案階段** | 專案目前進度 | 從「目標設定」定義的階段選一（見下方） |
| **N. 發票日期** | 開立發票日期 | YYYY-MM-DD |
| **O. 發票號碼** | 發票號碼 | 依公司格式 |
| **P. 收款狀態** | 款項是否已收 | 已收 / 留空（待收） |
| **Q. 收款日期** | 實際入帳日期 | YYYY-MM-DD |
| **R. 備註** | 補充說明 | 自由填寫 |
| **S. 最後更新** | 最後修改時間 | **自動產生，不需手動填寫** |

### ⚠️ 注意：一案多筆的填法

如果一個專案分多期收款（頭款 + 尾款），**每一期獨立一列**。例如：

| 名稱 | 分期 | 含稅金額 |
|---|---|---|
| ABC 合作案 | 頭款 | 150,000 |
| ABC 合作案 | 尾款 | 150,000 |

## 三、收入類型（revenueType）

| 類型 | 代表意義 | 何時選用 |
|---|---|---|
| **現金收入** | 已確認的實際營收 | 合約已簽、發票已開、款項已收或確定會收 |
| **預計現金收入** | 高度可能的營收 | 報價已提供且客戶有明確意願，但尚未全部完成 |
| **預估現金收入** | 尚在洽談中的潛在營收 | 正在評估、客戶尚未明確回覆 |
| **資源交換** | 非現金的合作方式 | 以服務/資源互換，不涉及金流 |

---

## 四、專案階段（projectStage）

專案階段定義在「目標設定」Sheet 的 H / I / J 欄，**未來可直接在 Sheet 新增或調整**。

### 目前的階段與計入業績規則

| 階段 | 排序 | 計入業績？ | 說明 |
|---|---|---|---|
| **需求溝通** | 1 | ❌ | 初期接觸、了解需求 |
| **簡報提案** | 2 | ❌ | 已進入提案階段 |
| **報價提供** | 3 | ✅ | 已提供正式報價 |
| **合作確認** | 4 | ✅ | 雙方確認合作意願（含資源交換） |
| **報價簽回** | 5 | ✅ | 客戶已簽回報價單 / 合約 |
| **結案** | 6 | ✅ | 專案完成，所有款項已收齊 |

### 如何在「目標設定」中管理階段

在「目標設定」Sheet 的 H、I、J 欄：
- **H 欄**：階段名稱
- **I 欄**：是否計入業績（填「是」或「否」）
- **J 欄**：排序數字（決定階段先後順序）

新增階段只要在這裡多加一列，儀表板會自動讀取並更新下拉選單。

## 五、KPI 計算邏輯

### 營收如何計算？

一筆專案要被計入 KPI，必須同時滿足兩個條件：

```
條件 1：專案階段的「計入業績」= 是（≥ 報價提供）
  AND
條件 2：收入類型 ≠ 資源交換
```

| 專案階段 | 收入類型 | 計入 KPI？ | 原因 |
|---|---|---|---|
| 報價提供 | 預估現金收入 | ✅ | 階段夠成熟，且非資源交換 |
| 報價簽回 | 現金收入 | ✅ | 已簽約確認 |
| 需求溝通 | 預估現金收入 | ❌ | 階段太早，不計入 |
| 合作確認 | 資源交換 | ❌ | 資源交換不計營收 |
| 簡報提案 | 現金收入 | ❌ | 階段太早（且此組合不合理） |

### KPI 分類邏輯

| KPI 類別 | 包含的收入類型 | 說明 |
|---|---|---|
| **確認營收** | 現金收入 | 已入袋或確定會入袋的金額 |
| **預估 Pipeline** | 預計現金收入 + 預估現金收入 | 尚在推進中的潛在營收 |
| **達成率** | 確認營收 ÷ 目標 | 僅看已確認的金額 |
| **含 Pipeline 達成率** | (確認 + Pipeline) ÷ 目標 | 樂觀預估 |

### 目標體系

- **目標 L**（保守）：基本應達標的水準
- **目標 H**（積極）：挑戰目標
- 目標值在「目標設定」Sheet 的 A–F 欄設定，按年度 + 季度

### 金額顯示

- Sheet 原始值為**未稅金額（元）**
- 儀表板顯示**含稅金額**（未稅 × 1.05）
- KPI 卡片以「萬」為單位顯示

## 六、收入類型與專案階段的合理對應

儀表板會在編輯時提供軟提示，當類型與階段不一致時顯示黃色警告：

| 專案階段 | 建議的收入類型 | 不建議 |
|---|---|---|
| 需求溝通 | 預估現金收入 | ❌ 現金收入、預計現金收入 |
| 簡報提案 | 預估現金收入 | ❌ 現金收入、預計現金收入 |
| 報價提供 | 預估 或 預計現金收入 | ❌ 現金收入 |
| 合作確認 | 預計 或 現金收入 | ❌ 預估現金收入 |
| 報價簽回 | 預計 或 現金收入 | ❌ 預估現金收入 |
| 結案 | 現金收入 | ❌ 預計、預估 |

**資源交換**可搭配任何階段。

## 七、填寫 Best Practice

### ✅ 正確做法

1. **一案一列、一期一列**：頭款尾款分開填，各自追蹤收款狀態
2. **階段即時更新**：案子推進時立即更新階段，保持 Pipeline 資料即時
3. **客戶名稱統一**：同一客戶永遠用同一名稱，樞紐分析才會正確歸類
4. **金額填未稅**：系統自動乘 1.05 算含稅，不要自己算
5. **報價提供以上一定要填金額**：系統會強制驗證
6. **月份填 0 代表待定**：比留空白更好，儀表板會顯示「待定」

### ❌ 常見錯誤

1. **一案只填一列但分多期收款**：會無法追蹤各期收款狀態
2. **階段和類型不一致**：例如階段已到「報價簽回」但類型還是「預估現金收入」
3. **客戶名稱不統一**：「ABC 公司」vs「ABC」vs「abc」會被當成三個不同客戶
4. **直接填含稅金額到未稅欄位**：會導致含稅金額多乘一次 1.05
5. **忘記更新階段**：案子已簽約但階段還停在「報價提供」，KPI 分類會失準

### 💡 進階技巧

- **樞紐分析**不受列表篩選器影響，永遠顯示全部資料
- **資源交換**可透過開關控制是否顯示在樞紐表中
- **近期更新**篩選器可快速找出最近 7 / 14 / 30 天有異動的案子
- **欄位顯示**可透過齒輪按鈕自訂要顯示哪些欄位

## 八、「目標設定」Sheet 總覽

「目標設定」是儀表板的設定中心，包含以下區塊：

| 欄位範圍 | 內容 | 用途 |
|---|---|---|
| A–F 欄 | 年度目標（L / H × 四季） | 計算達成率 |
| H–J 欄 | 專案階段定義 | 控制階段選項、計入業績規則、排序 |

修改這些設定後，儀表板下次載入就會自動套用，不需要改程式碼。

---

## 九、自動連動的項目

以下項目由系統自動處理，**不需手動維護**：

| 項目 | 觸發條件 | 說明 |
|---|---|---|
| 最後更新時間（S 欄） | 透過儀表板編輯時 | 自動寫入日期+時間 |
| 含稅金額計算 | 前端顯示時 | 未稅 × 1.05，Sheet 不需公式 |
| 季度歸屬 | 依月份自動分類 | 1–3月=Q1, 4–6月=Q2, 7–9月=Q3, 10–12月=Q4 |
| KPI 達成率 | 即時計算 | 依目標 L / H 自動計算百分比 |
| 專案階段下拉選項 | 讀取「目標設定」H–J 欄 | 新增階段後自動出現在儀表板 |
| 歷史年度灰色樣式 | 切換非當年度時 | 自動套用灰色 KPI 卡片 + 歷史年度標籤 |
