前置設定指南

讓 Claude Code 遠端管理 WordPress 網站 — 從設定到上線的完整步驟。

預估閱讀時間：15 分鐘 最後更新：2026-03-26
1

概述

WP-CLI 是 WordPress 的官方命令列工具，自 2012 年開始發展，至今已是非常成熟且穩定的生態系。它讓你可以用指令完成幾乎所有 WordPress 管理操作 — 安裝外掛、更新核心、管理使用者、匯出資料庫，全部都不需要打開瀏覽器。

將 WP-CLI 與 SSH 搭配，再加上 Claude Code（AI 工具），你就可以在一個終端機裡同時管理 5 個 WordPress 網站。只需要用自然語言描述你的需求，Claude Code 就能自動產生並執行對應的 WP-CLI 指令。

系統架構

Your Mac (Claude Code) | | SSH Tunnel | +------> Server 1 WP-CLI → WordPress Site A | +------> Server 2 WP-CLI → WordPress Site B | +------> Server 3 WP-CLI → WordPress Site C | +------> Server 4 WP-CLI → WordPress Site D | +------> Server 5 WP-CLI → WordPress Site E
2

工程師需要做的事（伺服器端）

以下步驟需要由有伺服器管理權限的工程師完成。每一台伺服器都需要執行一次。

2-1 安裝 WP-CLI（每台伺服器都要）

在每一台 WordPress 伺服器上執行以下指令，下載並安裝 WP-CLI：

bash 
# 下載 WP-CLI
curl -O https://raw.githubusercontent.com/wp-cli/builds/gh-pages/phar/wp-cli.phar

# 確認可以正常執行
php wp-cli.phar --info

# 設定執行權限並移到系統路徑
chmod +x wp-cli.phar
sudo mv wp-cli.phar /usr/local/bin/wp

# 驗證安裝成功
wp --version
前置需求：PHP 7.4 以上版本，且 WordPress 已安裝並可正常運作。
注意：如果 WordPress 安裝在非標準路徑，執行 WP-CLI 指令時請務必加上 --path=/your/wp/path 參數。

2-2 建立 SSH 使用者（建議）

建議建立一個專用的 AI 操作帳號（例如 llm），而非直接使用 root 帳號。命名為 llm 可以一眼辨識這是 AI 專用帳號。這個使用者需要有權限執行 WP-CLI 操作 WordPress 安裝目錄下的檔案。

bash 
# 建立 llm 使用者（AI 專用帳號）
sudo adduser llm

# 將使用者加入 www-data 群組，使其有權限操作 WordPress 檔案
sudo usermod -aG www-data llm

2-3 接收公鑰

工程師需要將使用者提供的公鑰新增到每台伺服器上 llm 帳號的 ~/.ssh/authorized_keys 檔案中。可以手動新增，也可以使用 ssh-copy-id 工具。

bash - 手動方式 
# 建立 .ssh 目錄
mkdir -p /home/llm/.ssh

# 將使用者提供的公鑰寫入 authorized_keys
echo "使用者提供的公鑰內容" >> /home/llm/.ssh/authorized_keys

# 設定正確的權限（非常重要）
chmod 700 /home/llm/.ssh
chmod 600 /home/llm/.ssh/authorized_keys
chown -R llm:llm /home/llm/.ssh

2-4 確認 WP-CLI 可正常運作

切換到 llm 使用者，進入 WordPress 根目錄，確認 WP-CLI 可以正常存取並操作 WordPress：

bash 
# 切換到 llm 使用者
su - llm

# 進入 WordPress 根目錄
cd /var/www/html

# 確認 WordPress 版本
wp core version

# 列出已安裝的外掛
wp plugin list
如果兩個指令都能正常輸出結果，表示伺服器端設定完成。請對所有 5 台伺服器重複以上步驟。
3

使用者需要做的事（Mac 端）

以下步驟由要使用 Claude Code 的人在自己的 Mac 上完成。

3-1 產生 SSH Key

SSH Key 是一組加密金鑰對，用來在你的 Mac 和伺服器之間建立安全連線，不需要輸入密碼。

bash 
ssh-keygen -t ed25519 -C "your-email@company.com"
# 按 Enter 三次（使用預設路徑、不設定密碼）

執行後會產生兩個檔案：

~/.ssh/id_ed25519
私鑰 — 留在你的 Mac 上，絕對不要分享給任何人。
~/.ssh/id_ed25519.pub
公鑰 — 交給工程師，讓他們放到伺服器上。

複製公鑰內容，傳給工程師：

bash 
cat ~/.ssh/id_ed25519.pub
# 複製輸出的內容，傳給工程師

3-2 設定 SSH Config

SSH Config 就像是伺服器的電話簿。設定好之後，你只要輸入短名稱（例如 site-a）就能連上對應的伺服器，不需要記住 IP 和連接埠。

~/.ssh/config 
# ~/.ssh/config

Host site-a
    HostName 123.456.789.001
    User llm
    IdentityFile ~/.ssh/id_ed25519
    Port 22

Host site-b
    HostName 123.456.789.002
    User llm
    IdentityFile ~/.ssh/id_ed25519
    Port 22

Host site-c
    HostName 123.456.789.003
    User llm
    IdentityFile ~/.ssh/id_ed25519
    Port 22

Host site-d
    HostName 123.456.789.004
    User llm
    IdentityFile ~/.ssh/id_ed25519
    Port 22

Host site-e
    HostName 123.456.789.005
    User llm
    IdentityFile ~/.ssh/id_ed25519
    Port 22
重要提醒：
  • 將 HostName 替換為實際的 IP 位址或網域名稱
  • 將 User 替換為實際的 SSH 使用者名稱
  • Port 通常是 22，如果不同請向工程師確認

3-3 測試連線

設定完成後，逐一測試每台伺服器的連線狀況：

bash 
# 測試 SSH 連線
ssh site-a "echo 'Connected!'"

# 測試 WP-CLI 是否可用
ssh site-a "cd /var/www/html && wp --version"

# 測試 WP-CLI 能否操作 WordPress
ssh site-a "cd /var/www/html && wp plugin list"
如果連線失敗，請檢查：
  • SSH 金鑰是否已正確新增到伺服器上？
  • 伺服器防火牆是否允許 SSH 連線？
  • WordPress 路徑是否正確？

安全注意事項

使用 SSH 連線管理 WordPress 時，請務必注意以下安全要點：

項目 說明
SSH 金鑰 私鑰絕對不離開你的 Mac，任何情況下都不要分享。只提供公鑰給工程師。
使用者權限 使用 llm 帳號（AI 專用），避免直接使用 root 帳號操作。
防火牆 如果條件允許，限制只有特定信任的 IP 位址可以透過 SSH 連線到伺服器。
WP-CLI 權限 SSH 使用者必須擁有正確的檔案讀寫權限，才能正常操作 WordPress 檔案。
稽核 定期檢查伺服器上的 authorized_keys 檔案，移除不再需要的公鑰。
密碼短語 如需更高安全性，可為 SSH 金鑰加上密碼短語（passphrase），但自動化操作時需搭配 ssh-agent 使用。
